Every session is a story

You open a session. The agent tries something bold; you ask it to roll back. It suggests restarting a service; you remind it not to kill system processes in prod. By Wednesday, you've typed the same guardrails three times.

Those corrections are hard‑won lessons—and they vanish when the chat ends.

Mnemaris makes them stick.

It reads your session history (messages, visible steps, outcomes), spots what you reject, what you accept, and the safer path you prefer. Then it drafts clear, specific instructions so the next session starts aligned. And if you're not ready to write files, run read‑only and review everything first.

Result: fewer detours, more momentum—the same you, just remembered.

How it works

1
Point to your history
Import your past sessions—start with Claude Code logs/transcripts; choose last sprint, last quarter, or all time.
2
We extract your patterns
  • Solutions you rejected (and the cues that predicted them)
  • Environment constraints (what never to restart or kill, guarded commands)
  • Communication preferences (diagnostic‑first, verbosity, code style)
  • Common rollbacks and the safer route that finally worked
3
Draft & apply (or just preview)
Get two outputs:
  • A portable core: AGENT_PLAYBOOK.md
  • A system profile: e.g., CLAUDE.md for Claude Code (today) and adapters for Codex / Gemini CLI assistant (starting line, rolling out via adapters)
Run insight‑only to see findings before you adopt them.

Example rules Mnemaris might generate

  • Prefer diagnostics first: logs/diffs/tests before any mutating action.
  • Do not kill system processes or restart databases without a snapshot/checkpoint.
  • Refactors: write/adjust unit tests first, then incremental changes.
  • Reply format: be concise; end with Next actions.
  • Incidents: propose a sandboxed, non‑destructive path before production changes.
